# מחירון תבליני דור זוארץ — אתר מוכן לחנות אונליין

עמוד HTML עברי (RTL) עם מחירון מלא, סל קניות, בחירה בין איסוף עצמי או משלוח, מינימום הזמנה למשלוח (₪150), שמירה ב‑localStorage ושליחת הזמנה ישירות ב‑WhatsApp. נבנה דאטה‑דריבן (קובץ JSON אחד) — קל להמשיך לחנות אינטרנטית מלאה.

## תוכן התיקייה

```
spice-menu/
├── index.html          ← העמוד הראשי
├── style.css           ← עיצוב, RTL, מובייל, סל, צ׳קאאוט, הדפסה
├── script.js           ← רינדור + סל + איסוף/משלוח + WhatsApp
├── data-inline.js      ← Fallback לפתיחה ב-file:// (מוטמע ב-window)
├── data/
│   └── products.json   ← מקור האמת — כל המוצרים והמחירים
├── assets/
│   ├── logo.png        ← לוגו תבליני דור זוארץ
│   ├── background.jpg  ← רקע (החנות בשוק נתניה)
│   └── badatz.png      ← חותמת בד״ץ בית יוסף
└── README.md
```

## איך פותחים את האתר מקומית

### אפשרות 1 — לחיצה כפולה על `index.html`
עובד מיידית בזכות `data-inline.js` שמוטמע כ‑fallback. אם ה‑fetch של `data/products.json` נחסם (כי זה `file://`), הקוד נופל לדאטה המוטמעת.

### אפשרות 2 — שרת מקומי (מומלץ)
בטרמינל, מתוך תיקיית `spice-menu`:

```bash
python3 -m http.server 8000
# או
npx serve .
# או
php -S localhost:8000
```

ואז: <http://localhost:8000>

> אחרי כל עריכה של `data/products.json` — לסנכרן את ה‑fallback של `file://`:
> ```bash
> python3 -c "import json; d=json.load(open('data/products.json',encoding='utf-8')); open('data-inline.js','w',encoding='utf-8').write('window.__PRODUCTS_DATA__='+json.dumps(d,ensure_ascii=False)+';')"
> ```

## איך עורכים מוצרים ומחירים

המקור היחיד הוא **`data/products.json`**.

```json
{
  "id": "p001-...",
  "name": "פפריקה מתוקה עם / בלי שמן",
  "category": "תבלינים",
  "description": "",
  "variants": [
    { "label": "200 גרם", "price": 10 },
    { "label": "500 גרם", "price": 20 },
    { "label": "קילו",   "price": 40 }
  ],
  "priceNote": null,
  "isWeightedProduct": true,
  "isUnitProduct": false,
  "isPromotion": false,
  "imagePlaceholder": true
}
```

### סוגי כרטיסי מוצר
- **כמה משקלים** — `variants` עם 2+ פריטים → כרטיס עם chips. המחיר וכפתור "הוספה לסל" משתנים לפי הבחירה.
- **משקל אחד** — `variants` עם פריט יחיד → **כרטיס קומפקטי**: שם, משקל, מחיר, כפתור סל (ללא בחירה).
- **מחיר לפי שקילה** — `variants: []` ו‑`priceNote: "מחיר לפי שקילה"` → תג בלבד, **בלי כפתור סל**.
- **מבצע** — `isPromotion: true` → תג "מבצע" + ה‑`priceNote` כטקסט (ללא חישוב מספרי).

### עריכה שכיחה
- **שינוי מחיר**: עורכים את `price` בתוך ה‑`variant` המתאים.
- **הוספת משקל**: מוסיפים פריט נוסף ל‑`variants`.
- **מוצר חדש**: אובייקט עם `id` ייחודי. סדר הקטגוריות מ‑`categoryOrder`.

לאחר עריכה — `Cmd/Ctrl+R` בדפדפן. שינוי מיידי.

## איך משתמשים בסל הקניות

1. **הוספה לסל** — בכל מוצר עם מחיר, לוחצים "הוספה לסל". הכרטיס מציג חיווי "נוסף לסל ✓".
2. **כפתור סל צף** בפינה התחתונה. תג צהוב מציג את כמות הפריטים.
3. **לחיצה על FAB** פותחת את הסל — drawer מצד הקריאה בדסקטופ, bottom‑sheet במובייל.
4. **בכל פריט בסל**: שם, משקל/יחידה, כמות (`+`/`−`), סכום ביניים, כפתור הסרה (`✕`).
5. **בתחתית הסל**: סה״כ כולל, אזור בחירה (איסוף/משלוח), שדות פרטים, כפתור שליחה לווצאפ, כפתור ניקוי סל.

> **חשוב**: בסיכום הסל ובהודעת ה‑WhatsApp **לא מוצג מחיר ליחידה**, רק שם, משקל, כמות, סה״כ לפריט, וסה״כ להזמנה.

### שמירה ב‑localStorage
הסל נשמר במפתח `dor-zoaretz-cart-v1`. רענון או סגירת הדפדפן לא מאבדים את הסל.

**פרטי לקוח (שם, טלפון, כתובת, הערות) לא נשמרים ב‑localStorage** — הם משמשים רק לבניית הודעת ה‑WhatsApp ומתאפסים בכל פתיחה מחדש של הסל.

איפוס מהיר (DevTools → Console):
```js
localStorage.removeItem("dor-zoaretz-cart-v1");
location.reload();
```

## איך עובד תהליך איסוף עצמי

1. הלקוח בוחר **"איסוף עצמי משוק נתניה"** ב־"איך תרצו לקבל את ההזמנה?".
2. נפתחים שדות חובה: **שם מלא**, **טלפון נייד**.
3. מוצג טקסט הסבר: "איסוף עצמי מהכתובת: היהלום 5, שוק נתניה. ניצור איתכם קשר לאישור ההזמנה ומועד האיסוף."
4. כפתור הסיום משתנה ל־**"שליחת הזמנה לאיסוף בווצאפ"**.
5. בלחיצה: ולידציה → אם תקין, נפתח WhatsApp עם הודעה מובנית. אם חסר משהו — מוצגות הודעות שגיאה אדומות ליד השדות.

## איך עובד תהליך משלוח

1. הלקוח בוחר **"משלוח"**.
2. אם סכום הסל **מתחת ל‑₪150** — מוצגת התראה אדומה: "מינימום הזמנה למשלוח הוא ₪150. חסרים לך עוד ₪X". כפתור השליחה מנוטרל. הלקוח עדיין יכול לעבור לאיסוף עצמי.
3. אם סכום הסל **₪150 ומעלה** — נפתחים שדות חובה: **שם מלא**, **טלפון**, **עיר**, **רחוב**, **מספר בית** + **הערות לשליח** (אופציונלי).
4. כפתור הסיום משתנה ל־**"שליחת הזמנה למשלוח בווצאפ"**.
5. בלחיצה: ולידציה → אם תקין, נפתח WhatsApp עם הודעה מובנית. דמי משלוח לא מתווספים אוטומטית — הסכום בהודעה הוא סכום המוצרים בלבד.

## איך עובד מינימום משלוח ₪150

הסכום מוגדר ב‑`script.js`, אובייקט `STORE`:
```js
const STORE = {
  ...
  minDelivery: 150,  // ₪
};
```

חישוב חי בכל לחיצה/שינוי:
- אם נבחר משלוח וסכום הסל < 150 → מוצגת התראה עם הסכום החסר, וכפתור השליחה מנוטרל.
- אם הסכום ≥ 150 → התראה נסתרת, מוצג טקסט עזר חיובי, וכפתור השליחה פעיל (כפוף לולידציה של שדות).

## איך נפתחת הודעת WhatsApp

לחיצה על כפתור הסיום פותחת `https://wa.me/972536000549?text=...` בכרטיסיה חדשה. הטקסט מקודד דרך `encodeURIComponent` כך שעברית, שורות חדשות וסימני ₪ עוברים תקין.

**לא נשלח אוטומטית** — תמיד נפתחת אצל הלקוח שיחת WhatsApp עם ההודעה מוכנה לשליחה.

## איפה משנים את פרטי החנות והמספר

ב‑`script.js`, אובייקט `STORE` בראש הקובץ:

```js
const STORE = {

  address: "היהלום 5, שוק נתניה",
  phone: "053-6000549",
  whatsappPhone: "972536000549", // ללא + או -

```

ולעדכן גם את הטקסט המוצג ב‑`index.html`:
- ה‑`<section class="store-info-bar">` למעלה (כתובת + מספר + קישור WhatsApp)
- ה‑`<p class="footer-contact">` בפוטר
- כתובת האיסוף בתוך `<p class="checkout-help">` של המתודה pickup

## פתיחה בלי localhost?

✅ כן. `data-inline.js` מטמיע את הדאטה ב‑`window.__PRODUCTS_DATA__`. ה‑script מנסה תחילה `fetch`, ואם נכשל — נופל לדאטה המוטמעת.

חשוב לרענן את `data-inline.js` אחרי כל עריכה של `data/products.json`.

## נגישות ומובייל

- **RTL מלא**, `<html dir="rtl" lang="he">`.
- **יעדי מגע ≥34px** (chips, ניווט, qty, פעולות סל, רדיו, שדות).
- **חיפוש דביק** במובייל. **ניווט קטגוריות אופקי** עם snap‑points.
- **גופן input 16px** למניעת זום אוטומטי ב‑iOS.
- **`100dvh`**, **safe‑area‑insets** לאיפון עם נץ׳.
- **focus‑visible** מודגש למקלדת.
- **`prefers-reduced-motion`** מבטל אנימציות.
- **`Esc` סוגר את הסל**. לחיצה על ה‑backdrop גם.

## מה הלאה כדי להפוך לחנות מלאה

1. **תמונות מוצר** ב‑`products.json` (`imageUrl`).
2. **API/Backend** במקום `fetch('data/products.json')`.
3. **תשלום** (Tranzila/Cardcom/Stripe) במקום WhatsApp.
4. **דשבורד הזמנות** (Supabase/Sheets/Airtable).
5. **משלוחים** — חישוב לפי כתובת, אינטגרציה עם HFD.
6. **חשבונית/קבלה** דיגיטלית.
7. **מועדון לקוחות** — Auth + היסטוריה.
8. **PWA** — `manifest.json` + Service Worker.

המבנה של הסל (`STATE.cart`, `Cart.add/remove/setQuantity`, אובייקט פריט) מתאים בדיוק לאינטגרציות האלו.

## טיפול ב‑assets חסרים

- ל‑`<img>` של הלוגו וה‑badatz יש `onerror="this.style.display='none'"` — אם חסר, האזור פשוט נעלם.
- אם `assets/background.jpg` לא נטען, נשאר רקע גרדיאנט בגווני שמנת/צהוב/חום.

## בעיות נפוצות

- **WhatsApp לא נפתח?** ודאו שאינכם בתוך iframe חסום. במובייל זה פותח את האפליקציה.
- **הסל ריק אחרי רענון?** `localStorage` חסום (גלישה פרטית). באנונימי הסל לא נשמר.
- **מחיר לא מתעדכן בבחירת משקל?** רענון קשה (`Cmd+Shift+R`).
- **שדה לא מתקבל?** ודאו שמילאתם את כל שדות החובה (סומנו ב‑`*`).
